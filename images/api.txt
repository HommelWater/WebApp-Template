# WebApp Template - Backend API overview

There are several routers attached in the main.py file that handle different categories of endpoints. They are: '/auth', '/users', '/files'.
Client side code (HTML/CSS/JS) can be found in the 'src/interface/' folder and subfolders.
The response format of the server is simple, it always responds with a json object with two fiels, the *type* field, and the *data* field. 
Server errors are always formatted the following: {"type":"failure", "data":{"notification":"<ERROR_MESSAGE>", "href":"<OPTIONAL_FORWARDING_HREF>"}}. 
This makes it easy to determine if the request was successful and to transmit error messages to the user/developer.

# Objects
>*username*: String name of the user.
>*code*: String *invite_code* or *TOTP_code*.
>*invite_code*: Code that allows someone to generate a new user account. 
The code consists of an invitee username and HOTP code, this code gets refreshed each time a new *invite_code* is used or created. 
Example: 'admin_438801'.
>*session_key*: Key consisting of a UUID and host. 
Example: '9fd12105-1236-4a93-9695-52ad384964bc@127.0.0.1:8000'.
>*TOTP_code*: 6-digit code generated by an authenticator application used to authenticate the user.
Example: '635738'.

# Database tables
The database contains a few tables by default. Their definitions are located in the 'src/database.py' file. It is recommended to expand this file for your application.

# /auth: Authentication
Source code can be found in the 'src/routers/auth_router.py' file.

ENDPOINTS:
> GET '/auth': Returns the main webpage.
> POST '/auth/': Takes in a *username* and *code*. Returns a *session_key* if the user is authenticated properly or a *TOTP_private_key* if an invite code is entered.
> POST '/auth/verify_session': Takes in a *session_key*. This function is mainly used by peers to authenticate a user session.

FUNCTIONS:
> 'get_user_and_session': Takes in a *session_key* and returns a *user* and *session* dictionary. This function will get users registered on peer servers and their associated session data via a http request.
> 'user_is_invalid': Takes in a *user* dictionary and an optional boolean for *allow_foreign* to allow or disallow foreign users. This function returns authentication error messages if the user is not properly authenticated. It is encouraged to expand this function to new user permissions and requirements.
> 'get_invite_code': Takes in a *user*. This function creates a new invite code for the given user and returns it.

# /users: Users
Source code can be found in the 'src/routers/user_router.py' file.

ENDPOINTS:
> GET '/users/': Returns the user overview page.
> POST '/users/: Takes in a *session_key* for verification and returns all users' *username*, *id*, *parent_id* and *creation_datetime* (which is an UNIX timestamp).

# /files: Files
All files in the 'src/interface/' directory are automatically hosted publicly. 
Private files are stored in the 'src/files/' folder and require a valid session token and user to be accessed.

ENDPOINTS:
> POST '/files/': Takes an optional *session_key*. Returns all files the user is allowed to view.
> POST '/files/upload': Takes in a form with a *file*, *session_key*, and boolean *is_public*. *is_public* indicates if the file is downloadable without authentication. Stores the file on the disk and returns the metadata to the uploader user.
> POST '/files/download/{id}': Takes an optional *session_key*. Returns a file download if the user is allowed to download it.
> GET '/files/download/{id}': Returns a file download, works via URL.
