## ūüĒĀ Server Response Format

All server responses follow a unified JSON structure:

```json
{
  "type": "<RESPONSE_TYPE>",
  "data": { ... }
}
```

### ‚ĚĆ Error Format

Server errors are always returned in the following format:

```json
{
  "type": "failure",
  "data": {
    "notification": "<ERROR_MESSAGE>",
    "href": "<OPTIONAL_FORWARDING_HREF>"
  }
}
```

This makes it easy for client applications to:

* Determine whether a request was successful
* Display error messages
* Optionally redirect users

# ūüď¶ Objects

### `username`

* Type: `String`
* Description: Name of the user

---

### `code`

* Type: `String`
* Description: Either an `invite_code` or `TOTP_code`

---

### `invite_code`

* Description: Code used to create a new user account
* Format: `<inviter_username>_<HOTP_code>`
* Regenerated each time it is used or newly created
* Example:

  ```
  admin_438801
  ```

---

### `session_key`

* Description: Session identifier
* Format: `<UUID>@<host>`
* Example:

  ```
  9fd12105-1236-4a93-9695-52ad384964bc@127.0.0.1:8000
  ```

---

### `TOTP_code`

* Description: 6-digit code generated by an authenticator app
* Example:

  ```
  635738
  ```

# ūüĒź `/auth` ‚Äď Authentication

**Source:** `src/routers/auth_router.py`

---

## Endpoints

### `GET /auth`

Returns the main authentication webpage.

---

### `POST /auth/`

**Input:**

* `username`
* `code` (invite_code or TOTP_code)

**Returns:**

* `session_key` (if authentication successful)
* `TOTP_private_key` (if invite code is entered)

---

### `POST /auth/verify_session`

**Input:**

* `session_key`

**Purpose:**

* Used by peer servers to verify user sessions.

---

## Internal Functions

### `get_user_and_session(session_key)`

* Returns:

  * `user` dictionary
  * `session` dictionary
* Can fetch users from peer servers via HTTP request.

---

### `user_is_invalid(user, allow_foreign=False)`

* Validates user authentication and permissions.
* Returns authentication error messages if invalid.
* `allow_foreign`:

  * `True`: Allows foreign users
  * `False`: Disallows foreign users
* Recommended to expand for additional permission logic.

---

### `get_invite_code(user)`

* Generates a new invite code for a given user.
* Returns the newly created invite code.

---

## Endpoints

### `GET /users/`

Returns the user overview page.

---

### `POST /users/`

**Input:**

* `session_key`

**Returns:**

* `username`
* `id`
* `parent_id`

# ūüďā `/files` ‚Äď File Management

## Public Files

All files located in:

```
src/interface/
```

are automatically hosted publicly.

---

## Private Files

Private files are stored in:

```
src/files/
```

These require:

* A valid `session_key`
* An authenticated user

---

## Endpoints

### `POST /files/`

**Input:**

* Optional `session_key`

**Returns:**

* List of files the user is authorized to view.

---

### `POST /files/upload`

**Form Data:**

* `file`
* `session_key`
* `is_public` (boolean)

**Behavior:**

* Stores the file on disk.
* `is_public` determines if authentication is required to download.
* Returns file metadata to the uploader.

---

### `POST /files/download/{id}`

**Input:**

* Optional `session_key`

**Returns:**

* File download if user has permission.

---

### `GET /files/download/{id}`

* Direct download via URL.
* Intended for public file access.
